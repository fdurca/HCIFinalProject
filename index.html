Executive Summary

Many systems and interactions - social networks, molecules, organizations, citations, physical models, transactions - can be represented quite naturally as graphs. How can we reason about and make predictions within these systems?

One idea is to look at tools that have worked well in other domains: neural networks have shown immense predictive power in a variety of learning tasks. However, neural networks have been traditionally used to operate on fixed-size and/or regular-structured inputs (such as sentences, images and video). This makes them unable to elegantly process graph-structured data.

Graph neural networks (GNNs) are a family of neural networks that can operate naturally on graph-structured data. By extracting and utilizing features from the underlying graph, GNNs can make more informed predictions about entities in these interactions, as compared to models that consider individual entities in isolation.

GNNs are not the only tools available to model graph-structured data: graph kernels and random-walk methods were some of the most popular ones. Today, however, GNNs have largely replaced these techniques because of their inherent flexibility to model the underlying systems better.

In this article, we will illustrate the challenges of computing over graphs, describe the origin and design of graph neural networks, and explore the most popular GNN variants in recent times. Particularly, we will see that many of these variants are composed of similar building blocks.

First, let's discuss some of the complications that graphs come with.

Navigational Map

Design Principles

Composite Granularity

User action for solitaire, sea battles, and spades has been modified to use composite granularity instead of atomic granularity. The option to operate with atomic granularity is still there.

Explicit Affordance

A big thing incorporated in this redesign is the use of explicit affordance. The home button and arrows for the shuffle function are all very intuitive in letting the user know the functionality behind the buttons. There are also small errors indicating a drop-down menu for the filter which also makes use of explicit affordance.

User Centered Design

As users of cardgames.io, we wanted to redesign the website and make changes that improve the site for the user. As users, we wanted to see more flexible interactions with the game and a cleaner site.

Built around making user interactions easier we added:

  • Keyboard integration for games
  • Touchpad friendly controls
  • Minimalist website design
  • Touchpad friendly controls

Hick’s Law

This redesign makes use of Hick’s law by limiting the number of options available to the user. Previously, every game was listed on the home page which caused information overloading but filters prevented that. The filter can be done by game type, difficulty, and players. Fewer options mean quicker decision-making for the user. There is also the shuffle option which allows the user to shuffle if they want to try a new game which limits the user to two options (left and right arrow).

Semiotics

Immediate Activation

The website takes advantage of immediate activation to inform the user about each game with a little description. This description activates once the user hovers over the game. Activation keeps information overload down but provides documentation to the user.

Discrete Flow

With the added one-click functionality in Spider Solitaire, the user is now able to utilize discrete flow through instantaneous actions, while still being able to use the drag controls to demonstrate continuous flow.

Fitts’ Law

By adding keyboard control to Spades, the user can start the game by pressing the enter key situated right next to their right pinky. This greatly reduces the time to start the game because the user is required to move their cursor to the “Enter” button

Positive Transfer

The redesign makes use of positive transfer when allowing the user once click action in solitaire. When investigating other solitaire games, the team realized that one-click action is a common feature among solitaire games. Users will naturally interact with the game using the one-click feature because of positive transfer from other card games.

Node-Order Equivariance

The original idea to edit cardgames.io was because Filip, Ryan, and some of their friends were consistent users of cardgames.io and felt like it could be improved to reflect HCI’s best practices. While coming up with tasks and fixes to cardgames.io, the team interviewed Filip's and Ryan’s friends to determine what they would like to see modified. These friends were later consulted to get feedback on the redesign. As members who gave their input throughout the project, this project made use of participatory design.

Formative Evaluation

To complete the redesign using an agile methodology, the team used formative evaluation throughout the project. This evaluation was done by team members (user-centered design) and outside members (participatory design). A summative evaluation was done after the implementation of the project.

Mental Model Activation

Adding the home button to every game makes use of mental model activation. The home button appearing as a house is a metaphor for someone’s home in the real world. Since people’s homes are the main location for all their belongings, the home button taking the user to the location of all the games activates the user’s mental model. The home button also makes use of positive transfer since most apps have a home button to return to the main area of the map.

Responsive Design

Making the site responsive and adjusting the UI for various screen sizes.

Mix Mode Interface

The original site makes use of direct manipulation interfaces in the games because a pointing device is used to manipulate representations directly. In the redesign, the users can also use their keyboard to play sea battles and spades which is indirect interface manipulation. Both these manipulations together make the interface a mixed-mode interface.

Diversity Macro Level Interaction

Part of the goal of the redesign was to make the games more accessible to all users. Adding keyboard support and one-click action allows users who are not able to drag for any reason to still be able to play the game. Having all users be able to play the games is making use of macro-level interaction.

Consistency

Given the redesigns of various pages, the overall design scheme stayed consistent, and key functions, features, and icons are kept consistent throughout the site.

Vertical Prototyping Functionality

While doing the redesign, the team added vertical prototyping functionality to the site. This means adding in-depth functionality for only a few selected features. The redesigned site included many interactive changes to a few games on the site (3) instead of minor changes to every game on the site.

Local Constraint

Added to the redesign was a game filtering system that allows the user to filter games based on difficulty, number of players, and type of game. This makes use of logical constraints because it reduces the number of options that are not relevant to the user. If a user wants to play a card game involving 4 players, they can filter which prevents them from selecting a game with only 2 players.

Visibility

Using keyboard controls in sea battles meant that we needed to let the user know which grid cell the user was currently selecting. We fixed this by adding a reticle that hovers over the grid cell that the user is currently selecting. We need to do the same thing with the game spades to inform the user about the visibility of the system. Now when the user hovers over a card, the card displays a gold border and appears higher than the other cards. Both these changes are crucial to let the user know what is going on in the system.

Heuristic

When taking a look at the heuristic evaluation of the redesigned website, the following improvements have been made:

  • User Control & Freedom
  • Error Prevention
  • Flexibility and Efficiency of Use
  • Aesthetic and Minimalist Design
  • Help & Documentation

User Control & Freedom

Our redesigned site includes a home button that allows the user to return to the main menu where they can select a game of their choosing. In the original version of the site, the user could click the drop-down menu titled “More Games” which displayed a list of all the games on the site but it was difficult to read and very congested. The implementation of the home button gives the user more control over the site. There was also the implementation of a shuffle feature that allows the user to switch to different games by clicking arrows on the right and left sides of the game. This is a metaphor for the shuffle function on a music platform where the user can try different games. This is another feature that gives the user more freedom over the website.

Error Prevention

One of the biggest problems with the original site was the information overload on the main screen. Displayed are 42 different game icons with no explanation about any of the games. To improve, we decided to implement a game filtering feature so that the user can filter games based on difficulty, number of players, and type of game. This will prevent the user from making errors by using constraints, for example, if the user wants to play a card game they can easily filter for those games while other games remain hidden. This prevents the user from playing a game that they did not want to originally play.

Flexibility and Efficiency of Use

The biggest contribution to our redesign was implementing efficiency and flexibility. As we were avid users of the website, there was a heavy focus on user-centered design and flexibility was a feature that was lacking. In spider solitaire, the only way to move a card was to click and drag which is a difficult way to play for laptop users. The feature that was added was the ability to use one-click action to select a card and then use one-click action to place that card on a selected pile. This gives the user more flexibility because the user can drag and drop but also click on both card piles. This feature is also more efficient for laptop users because clicking is quicker and easier than dragging. The other feature that was added was keyboard support for the game sea battles and Spades. Both these games do not have keyboard support so laptop users must use the trackpad to play the game. The keyboard support allows the user to play both games only using only the keyboard which is quicker, easier, and more comfortable for laptop users. Overall these 3 features make the gameplay much more enjoyable.

Aesthetic and Minimalist Design

Another big improvement to the site was a more minimalist and aesthetically pleasing design by using a minimalist and modern design language. This meant using a black-and-white color scheme and using a flat design for the icons. There are also fewer icons on the main menu due to filtering and the icons are bigger and have a more simple design. This makes the overall site look more aesthetically pleasing and puts less memory overload on the user.

Help & Documentation

Finally, this site included documentation about each game at the bottom of each game’s page but there is no information about the games on the main menu. That means if the user wanted to know how to play a game they would be required to click and navigate to that game’s page. We added a hover feature where when the user hovers over a game icon, a text box appears giving a short description informing the user of how the game works. This makes use of immediate activation by the text box popping up on hover and keeps information overload down.

Recommendations

In light of our final evaluation for the redesign, there are still improvements that can be made to the site. Theses improvements are as follows:

  • Implementing features in more games

    Currently the cardgames.io site has 42 games which could all use improvements in different ways. Some of these improvements can be keyboard support and one click action which we have previously talked about. This would allow the entire site to function without using a trackpad which would make the site more efficient, flexible and usable for all. There could also be other features implemented to other games as well.

  • Developing more keyboard support for sea battle

    Currently, keyboard support for sea battles allows users to attack ships while just using the keyboard. Placing ships on the other hand can only be done with the mouse. This included rotating ships as well. We would like to add keyboard support for this as well so the entire game can be played with the keyboard adding more flexibility, efficiency to usability for all users.

  • In-game tutorials

    Another feature would be adding in-game tutorials to each game on the site. These tutorials would be on the game page for each game. This would help inform the user and provide a better idea of how to play the game. This is more clear and helpful than providing documentation.
